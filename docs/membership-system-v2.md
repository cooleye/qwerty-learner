# 会员系统 V2 —— 会员类型管理与激活码体系

## 一、概述

### 目标
1. 后台可配置多种会员类型（月度/年度/终身）
2. 生成激活码时选择类型，激活码带前缀标识
3. 用户激活时根据类型累加有效期
4. 前台 VIP/终身会员隐藏激活码兑换框
5. 激活码有完整生命周期：未使用 → 已售出 → 已使用
6. 用户管理支持禁用/启用，禁用用户失去会员资格
7. 发货弹窗方便线下销售
8. 激活码列表支持筛选、单行复制

### 名词定义
| 术语 | 说明 |
|------|------|
| **会员类型** | 月度会员、年度会员、终身会员等，定义不同的时长 |
| **激活码** | 一串带前缀的编码，关联到某个会员类型 |
| **有效期累加** | 激活后在原有剩余时长上追加，非覆盖 |
| **激活码状态** | unused（未使用）→ sold（已售出）→ used（已使用）|

## 二、数据库设计

### activation_codes 表（原地修改）

```sql
-- 在现有 activation_codes 表上增加字段
ALTER TABLE public.activation_codes ADD COLUMN membership_type TEXT NOT NULL DEFAULT 'yearly';
ALTER TABLE public.activation_codes ADD COLUMN duration_days INTEGER NOT NULL DEFAULT 365;
```

当前 activation_codes 表已有字段：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| code | TEXT UNIQUE | 激活码（带前缀，如 YEAR-xxxx） |
| batch | TEXT | 批次 |
| status | TEXT | unused（未使用）/ sold（已售出）/ used（已使用） |
| source | TEXT | admin（管理端生成）/ payment（支付购买） |
| used_by | UUID | 使用用户 ID |
| used_at | TIMESTAMPTZ | 使用时间 |
| created_by | TEXT | 创建者 |
| created_at | TIMESTAMPTZ | 创建时间 |
| **membership_type** | TEXT | 会员类型标识（monthly / yearly / lifetime）← 新增 |
| **duration_days** | INTEGER | 有效期天数 ← 新增 |

### 新增 membership_types 表

```sql
CREATE TABLE public.membership_types (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,              -- 显示名称（如"月度会员"）
  code TEXT UNIQUE NOT NULL,       -- 标识码（monthly / yearly / lifetime）
  prefix TEXT UNIQUE NOT NULL,     -- 激活码前缀（MON / YEAR / LIFE）
  duration_days INTEGER NOT NULL,  -- 有效期天数（月度=30，年度=365，终身=36135≈99年）
  price INTEGER,                   -- 价格（分），可选
  sort_order INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 插入默认类型
INSERT INTO public.membership_types (name, code, prefix, duration_days, sort_order) VALUES
  ('月度会员', 'monthly', 'MON', 30, 1),
  ('年度会员', 'yearly', 'YEAR', 365, 2),
  ('终身会员', 'lifetime', 'LIFE', 36135, 3);
```

### 3.5 激活逻辑

**有效期计算规则（累加）：**

```
新到期时间 = 
  - 终身会员类型 → now + 99年
  - 累加类型（月度/年度）→ max(now + durationDays, 当前到期时间 + durationDays)
```

**场景举例：**

| 当前状态 | 剩余天数 | 激活类型 | 激活后有效期 |
|---------|---------|---------|------------|
| 免费用户 | 0天 | 月度会员(30天) | 30天后 |
| VIP 用户 | 还剩10天 | 月度会员(30天) | 40天后（累加） |
| VIP 用户 | 还剩20天 | 年度会员(365天) | 385天后（累加） |
| 任意用户 | 任意 | 终身会员 | 99年后 |
| 终身会员 | 99年 | 任意类型 | 终身（输入框已隐藏） |

### 3.6 激活完成后

- 前端：隐藏激活码输入框，只展示会员状态和到期时间
- 后端：更新 `profiles.membership` 和 `membership_expire_at`，更新激活码状态为 `used`

## 五、API 变更

### 新增 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/membership-types` | 获取所有会员类型 |
| POST | `/api/admin/membership-types` | 新增会员类型 |
| PUT | `/api/admin/membership-types` | 编辑会员类型 |
| DELETE | `/api/admin/membership-types/:id` | 删除会员类型 |
| PUT | `/api/admin/codes/:id/sold` | 标记激活码为已售出 |
| PUT | `/api/admin/users/:id/disable` | 禁用用户 |
| PUT | `/api/admin/users/:id/enable` | 启用用户 |
| GET | `/api/admin/users/:id` | 获取用户详情（含激活记录） |

### 现有 API 变更

| API | 变更 |
|-----|------|
| `POST /api/admin/codes` | 增加 `membershipType` 参数，生成的激活码带对应前缀 |
| `GET /api/admin/users` | 增加 `page` `search` 分页参数 |
| `POST /api/admin/codes` | 返回激活码列表包含 membership_type、duration_days |

### Supabase RPC 更新

`redeem_activation_code` 函数更新逻辑：
1. 读取激活码的 `membership_type` 和 `duration_days`
2. 判断用户状态：如果 `status = 'disabled'` 拒绝激活
3. 判断激活码状态：如果 `status != 'unused' and status != 'sold'` 拒绝激活
4. 如果是 `lifetime` 类型 → `membership_expire_at = NOW() + 99年`
5. 如果是累加类型 → 取 `max(NOW(), 当前到期时间) + duration_days`
6. 更新 `profiles.membership = 'vip'`，更新 `expire_at`
7. 更新激活码 `status = 'used'`

## 六、用户禁用逻辑

### 前端权限校验

在 `DictDetail/index.tsx` 的 `onChangeChapter` 中增加校验：

```tsx
if (user?.status === 'disabled') {
  setShowDisabledTip(true)
  return
}
```

### 登录时校验

在 `Login/index.tsx` 中，登录成功后检查用户 profile：
```tsx
if (profile?.status === 'disabled') {
  await supabase.auth.signOut()
  setError('账号已被禁用，请联系管理员')
  return
}
```

### 被禁用的效果

- 无法登录
- 无法访问任何收费课程
- 失去所有会员权益
- 数据保留，启用后恢复

## 八、注意事项

1. **终身会员**：激活后有效期设为 99 年，不再显示激活码输入框
2. **激活码状态流转**：`unused → sold → used`（不可逆）
3. **禁用用户**：禁用后立即失效，所有课程均不可访问
4. **启用用户**：恢复会员权益，到期时间不变
5. **前缀唯一性**：每个会员类型的激活码前缀必须唯一
6. **已使用的会员类型**：不可删除，防止数据不一致
7. **发货弹窗**：提供复制按钮，方便客服人员复制信息发给客户
8. **到期提示**：仅对 VIP 会员生效（终身会员不需要）
